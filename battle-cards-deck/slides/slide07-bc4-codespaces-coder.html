BATTLE CARDS 4 & 5

vs. GitHub Codespaces & Coder

THREAT LEVEL: MEDIUM

GitHub Codespaces

GitHub owns the machine. Your existing servers don't exist in Codespaces world.

Costs escalate fast — 4-core × 8hr/day ≈ $300/month

Hard GitHub lock-in — useless for GitLab, Bitbucket, Azure DevOps shops

No terminal I/O streaming to external dashboard, no fleet view, no SSH key mgmt

Codespaces counter

"Codespaces spins up GitHub's machines for you. SessionForge manages Claude Code on machines you already own. If you have a $6/month VPS running Claude Code, Codespaces has nothing to offer you."

Coder

Requires DevOps team + Terraform expertise to deploy and maintain

Workspace-centric, not session-centric — no concept of "Claude Code process running now"

No real-time terminal I/O streaming, no CPU/memory monitoring per session

Provisioning-heavy — overkill for "I have machines, I want to see what's running"

Coder counter

"Coder is right if you're building a standardized dev environment platform from scratch with Terraform. SessionForge is right if you already have machines running Claude Code and need operational visibility today — no Terraform, no DevOps overhead. Install the agent binary, connect, done."

Objections — Both

"We're all-in on GitHub ecosystem"

Keep GitHub for your code. SessionForge manages where Claude Code runs — your infrastructure, not GitHub's.

"Coder is free and open source"

Coder server still needs to run somewhere and be maintained. SessionForge free tier is just agent + dashboard, zero ops overhead.

"Coder has AI agent governance"

Coder governs environments. SessionForge governs running sessions — live status, resource usage, start/stop. Different layers.

"We need RBAC and audit logs"

Our Team and Enterprise plans include both. Coder Premium is custom/expensive for features SessionForge covers at $49/month.
