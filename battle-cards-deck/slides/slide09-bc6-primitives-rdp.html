BATTLE CARDS 9 & 10

vs. tmate / ngrok & TeamViewer / AnyDesk

THREAT LEVEL: LOW

tmate / ngrok / ttyd — "Primitives"

These are primitives, not products — purely CLI, no dashboard

One terminal at a time — no fleet management

ngrok: 2-hour session limit, 1 GB bandwidth, ephemeral URLs on free tier

No session lifecycle management, no SSH key management, no audit logs

No CPU/memory monitoring — just a pipe, not a platform

Primitives counter

"tmate answers 'how do I show someone my terminal right now.' SessionForge answers 'how do I manage all my Claude Code sessions across all my machines from a browser without SSHing into each one.' Primitives vs. fleet platform."

Bonus: ngrok can actually be a connectivity layer under SessionForge for machines behind NAT. Position as complementary, not competitive — we ride on ngrok where needed.

TeamViewer / AnyDesk / Parsec — "Wrong Shape"

GUI remote desktop is wrong for headless server + Claude Code management

$25–230/month for functionality that doesn't match the problem

IT asset-centric fleet (device inventory, OS versions) — not developer session-centric

No terminal I/O streaming, no Claude Code awareness, no session log capture

AnyDesk Oct 2025 license change creating customer confusion and frustration

TeamViewer aggressively flags free accounts as commercial

Remote desktop counter

"TeamViewer shows you a desktop. You don't need a desktop — you need Claude Code's terminal output, know if the process is alive, and hit restart when it crashes. It's like using a tow truck to move a bicycle."

Objection Handling

"IT already rolled out TeamViewer company-wide"

Keep TeamViewer for IT support. SessionForge is developer tooling — separate buying decision, different budget, different buyer.

"I just SSH in when I need to check"

Works perfectly for one machine. SessionForge pays off the moment you have more than one. What's your time worth per SSH session × how many machines × how many days?
